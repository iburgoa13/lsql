Presentación

LearnSQL es un juez automático para el aprendizaje del lenguaje SQL. Aquí podrás enfrentarte a diferentes ejercicios, enviar una solución en código SQL y recibir automáticamente un veredicto incluyendo información del motivo del fallo en caso de que la solución enviada no sea correcta. El juez permite practicar con distintos tipos de problemas, como por ejemplo realizar consultas SQL, insertar/actualizar/modificar filas en las tablas o definir funciones y procedimientos. Internamente utiliza Oracle 11g para corregir los envíos recibidos.

Uso del juez

Para acceder al juez es necesario autenticarse con el usuario y contraseña que os habrá proporcionado vuestro profesor. Las principales funciones del juez aparecen en el menú situado en la parte superior de la ventana:

  • Ejercicios: muestra todas las colecciones de problemas disponibles
  • Mis envíos: muestra todos los envíos que has realizado al juez, pudiendo consultar su código enviado y el veredicto recibido.
  • Clasificación: permite visualizar la clasificación de los estudiantes dentro de tu clase en cada una de las colecciones. Puedes encontrar más información en la sección Clasificación.
  • Ayuda: muestra esta página de ayuda.
  • El menú desplegable situado en la parte derecha contiene opciones para cambiar tu contraseña, visualizar qué logros has obtenido (ver sección Logros) y cerrar la sesión.

El enunciado consta de una explicación textual del problema a resolver, seguida de una descripción del esquema de la base de datos inicial sobre la que se debe resolver el problema (apartado Base de datos). Además del esquema, este apartado incluye datos iniciales para entender mejor el contenido de la base de datos y qué es lo que se espera en dicho problema. Mediante el botón "Descargar Script" es posible descargarse todas las sentencias SQL necesarias para construir esa base de datos inicial en otro sistema y realizar consultas de prueba en tu propia instalación local de Oracle.

El apartado Resultado esperado contiene los resultados que debería generar una solución para considerarse correcta sobre la base de datos con los valores iniciales mostrados en el enunciado. A continuación aparece un cuadro donde se puede introducir el código SQL de la solución. También es posible subir una solución escrita previamente en un fichero SQL arrastrando o seleccionando dicho fichero en la zona de ficheros que se encuentra sobre el cuadro de código.

Cuando se pulsa el botón Enviar solución se envía al juez la solución que hay en el cuadro de código y se corrige. El juez puede contestar con distintos veredictos posibles:

  • Aceptado

    Tu código ha producido los resultados esperados en todos los tests probados. Se muestra una ventana flotante tal y como se ve en la siguiente figura:

  • Error en ejecución

    El juez ha intentado ejecutar el código SQL pero no ha podido porque ha lanzado algún error. Por ejemplo, este error aparece cuando hay errores sintácticos en alguna sentencia SQL, se accede a tablas o columnas que no existen, o cuando se insertan valores duplicados en una clave primaria. En estos casos el juez detecta en qué fila y columna de la consulta SQL se ha producido este error, lo remarca en el cuadro de código y además muestra un mensaje con los detalles del error.   • Resultados incorrectos

    Este veredicto aparece cuando el código SQL se ha podido ejecutar sin problema pero el resultado no es el esperado. En este caso la retroalimentación que aparece bajo el cuadro de código contiene información detallada sobre qué resultado ha generado tu código SQL y qué resultado se esperaba, tratando de indicar qué elementos faltan y cuáles sobran.   • Error de validación

    Si recibes un mensaje de este tipo significa que el código enviado no cumple con los requisitos necesarios y ni siquiera se ha llegado a ejecutar. Cada problema puede tener requisitos diferentes, por ejemplo, los problemas que solicitan consultas SELECT esperan que se envíe exactamente una sentencia SQL, mientras que otros problemas centrados en manipulación de tablas pueden requerir un número concreto de sentencias.   • Tiempo límite excedido

    Si el código que envías tarda demasiado tiempo en ejecutarse, el juez abortará su ejecución y te informará de ello. Este tipo de mensaje aparece cuando se envía código SQL extremadamente ineficiente. En estos casos deberás revisar tu solución y modificarla para que sea más eficiente.

  • Error interno

    Este tipo de error indica que ha ocurrido un error inesperado durante la corrección de tu envío. Es un error achacable al propio juez y no al código que has enviado. En ocasiones este error se debe una sobrecarga puntual del servidor y se corrige con el tiempo, pero si persiste debes informar a tu profesor.

Clasificación

Por cada colección de problemas se puede visualizar la clasificación de los participantes dentro de tu mismo grupo de clase. La clasificación muestra el listado ordenado de usuarios junto con información de cada ejercicio:

La primera columna muestra la posición del usuario, donde varios usuarios pueden compartir la misma posición si empatan en todos los aspectos. A continuación aparece el nombre de usuario junto con un icono indicando el número de logros que ha obtenido hasta el momento actual (ver sección Logros). A continuación aparece una columna por cada ejercicio de la colección. En cada celda aparece una descripción X/Y (Z) de los envíos realizados:

  • X: número de envíos correctos.
  • Y: número de envíos totales.
  • Z: posición del primer envío correcto (para el caso en que envíes más de una respuesta correcta). Si no has enviado ningún envío correcto todavía, este número coincidirá con el número de envíos totales.

Además, cada celda tiene un código de colores: verde si el usuario ha conseguido resolver ese ejercicio, rojo si el usuario ha realizado algún envío pero no ha conseguido resolver ese ejercicio, y blanco si el usuario todavía no ha realizado ningún envío a ese ejercicio.

Para ordenar a los usuarios en la clasificación primero se tiene en cuenta el número de ejercicios resueltos en esa colección, es decir, aparecerán antes aquellos usuarios con más ejercicios resueltos. En caso de empate, se desempatará penalizando el número de envíos necesarios para resolver los ejercicios (valor Z de las celdas). Para ello, la columna Puntuación suma el número de envíos que ha necesitado el usuario para resolver los ejercicios. Hay que tener en cuenta que el número de envíos realizados en los ejercicios no superados no se tiene en cuenta hasta que se resuelve el ejercicio, y por eso Soledad en la figura anterior suma una puntuación de 14, es decir, no se contabilizan sus 6 envíos fallidos al ejercicio 2. En resumen, el criterio para ordenar participantes es el siguiente:

  1. De mayor a menor número de problemas resueltos.
  2. En caso de empate, de menor a mayor puntuación (una menor puntuación indica menos envíos fallidos hasta resolver los problemas).
  3. En caso de empate, ocupan la misma posición numérica.
